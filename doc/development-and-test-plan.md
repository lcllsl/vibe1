# 3D 多面体贪吃蛇开发与测试计划

## 1. 依赖

运行时依赖：

```bash
npm install three vue-router@4
```

推荐测试依赖：

```bash
npm install -D vitest jsdom @vue/test-utils
```

可选动画依赖：

```bash
npm install gsap
```

`gsap` 可用于摄像机转场、护栏升降和食物动画。MVP 可以先不安装，使用 Three.js 和手写插值完成基础动画。

## 3. 开发约束

- 每次开发前先阅读 `doc/requirements.md` 和本文件。
- 游戏规则必须优先写成纯 TypeScript 模块。
- 不要把核心规则直接塞进 Vue 组件或 Three.js 渲染类。
- Three.js 只负责表现和输入输出连接，不拥有核心游戏规则。
- 涉及移动、碰撞、食物、效果、关卡的改动应补充或更新 Vitest 测试。
- 重大规则变更需要同步更新 `doc/requirements.md`。
- 浏览器验收需要检查画布非空、控制台无 error、交互可用。

## 4. Agents 和 Skill 策略

当前阶段不强制创建自定义 skill。先使用文档约束和分 session prompt 控制开发方向。

如后续开发中出现重复流程或多人/多 agent 协作，再创建项目专用 skill。该 skill 应至少约束：

- 开发前读取需求和计划文档。
- 核心规则保持纯逻辑模块化。
- Three.js 对象需要释放资源。
- 核心规则变更要补测试。
- 前端变更后要进行浏览器验收。

推荐 agents 划分：

- `Game Logic Agent`：移动、碰撞、食物、效果、关卡和单元测试。
- `Three Renderer Agent`：Three.js 场景、模型、护栏、食物、相机和动画。
- `Vue UI Agent`：设置页、游戏页、HUD、路由和页面状态。
- `QA Agent`：构建、测试、浏览器验收、性能和控制台检查。

不建议一开始并行启动太多 agents。Phase 1 由主线完成基础架构和可玩原型后，再按模块拆分更稳。

## 5. Session 区分策略

按阶段和关注点拆分 session，避免一个超长上下文承载全部开发。

- Session 1：Phase 1 单面原型。
- Session 2：Phase 2 立方体跨面。
- Session 3：Phase 3 食物和效果。
- Session 4：Phase 4 设置页和 HUD。
- Session 5：Phase 5 美术和手感。

每个 session 开始时：

- 阅读 `doc/requirements.md`。
- 阅读 `doc/development-and-test-plan.md`。
- 阅读 `doc/prompt.md` 中对应 session prompt。
- 检查当前 Git 状态。
- 检查已有代码结构。

每个 session 结束时：

- 运行可行的测试和构建。
- 做必要的浏览器验收。
- 总结完成项、风险和下一步。
- 如规则变化，更新文档。

## 6. 开发阶段

### Phase 1：单面可玩原型

目标：玩家可以在一个面上自由移动、吃食物、变长、失败。

任务：

- 安装必要依赖。
- 添加 Three.js 场景初始化。
- 渲染单个活动面。
- 实现连续移动。
- 实现自动前进、`A/D` 转向、`W` 加速。
- 实现蛇身体历史轨迹跟随。
- 生成普通食物。
- 检测食物碰撞。
- 检测自身碰撞，并忽略头部后方前 `20%` 身体长度。
- 添加暂停、重新开始、失败状态。

验收标准：

- 画布渲染非空 3D 场景。
- 蛇可以平滑移动。
- `A/D` 转向可用，`W` 加速可用。
- 吃普通食物后长度增加。
- 撞到自身后失败。
- `npm run build` 通过。

### Phase 2：立方体拓扑和跨面

目标：蛇可以在立方体六个面之间移动，已完成面永久关闭。

任务：

- 定义立方体 6 个面、边、法线和邻接关系。
- 实现当前面局部连续坐标。
- 实现从开放边跨入相邻面。
- 实现跨面后的坐标和朝向映射。
- 添加护栏系统。
- 当前长度达到本关目标长度后，降下可用出口护栏。
- 进入新面后，将上一面标记为 `completed`。
- 已完成面护栏永久升起。
- 添加死胡同状态表现，但不立即判负。
- 添加胜利检测。

验收标准：

- 玩家能通过开放边进入相邻面。
- 上一面永久关闭。
- 撞向已完成面会死亡，彩虹糖护盾除外。
- 完成立方体 6 个面后胜利。

### Phase 3：完整食物和效果系统

目标：实现 7 种食物和全部计时效果。

任务：

- 实现权重随机食物生成。
- 实现普通食物长度增长。
- 实现大餐长度增长。
- 实现加速 10 秒。
- 实现减速 10 秒。
- 实现集束爆开生成食物。
- 实现彩虹糖护盾。
- 实现超级药水。
- 实现临时食物 5 秒过期。
- 确保超级药水生成的临时食物不能是超级药水。
- 确保超级药水不影响已完成面护栏。

验收标准：

- 大样本下食物出现比例接近配置权重。
- 计时效果能正确结束。
- 临时食物能正确清理。
- 超级药水期间所有非 completed 面可生成临时食物。
- 超级药水不会打开 completed 面。

### Phase 4：摄像机、UI 和设置页

目标：让游戏可理解、可配置、可完整操作。

任务：

- 添加俯视跟随摄像机。
- 跨面时平滑旋转摄像机。
- 添加设置页。
- 添加设置页和游戏页路由。
- 显示当前关卡、当前面、速度、长度、目标长度和 active effects。
- 添加暂停和重新开始控制。
- 添加调试信息开关。
- 其他多面体选项显示为不可用或 `coming soon`。

验收标准：

- 活动面在移动中始终清晰。
- 跨面转场不让玩家迷失方向。
- 设置参数影响新游戏。
- HUD 状态和游戏状态一致。

### Phase 5：可爱视觉和手感优化

目标：让 MVP 看起来完整、柔和、可爱。

任务：

- 用可爱低多边形视觉替换占位形状。
- 应用柔和糖果色调。
- 添加圆润蛇身。
- 让 7 种食物在颜色和轮廓上容易区分。
- 添加护栏升降动画。
- 添加彩虹糖和超级药水的柔和粒子或发光效果。
- 添加轻盈的胜利和失败反馈。

验收标准：

- 食物类型视觉可区分。
- 色彩鲜艳但不刺眼。
- 重要游戏对象易读。
- 超级药水期间性能稳定。

## 7. 单元测试计划

单元测试优先覆盖纯逻辑，避免依赖 Three.js。

### 移动

- 自动前进会沿当前朝向推进蛇头。
- `A` 会向左旋转朝向。
- `D` 会向右旋转朝向。
- `W` 会临时提高速度。
- 移动基于 delta time，帧率变化不应改变结果。
- 速度倍率会影响移动距离。
- 身体段能按历史轨迹跟随。

### 碰撞

- 蛇头和食物的距离碰撞有效。
- 蛇头和身体碰撞有效。
- 身体碰撞忽略头部后方前 `20%`。
- 撞到升起护栏会死亡。
- 穿过开放边会进入预期相邻面。
- 撞到 completed 面护栏会死亡。
- 彩虹糖护盾能抵消一次致命碰撞。
- 彩虹糖不能让蛇进入 completed 面。

### 食物

- 权重随机生成符合配置概率。
- 普通食物增加长度。
- 大餐比普通食物增加更多长度。
- 集束生成多个随机食物。
- 集束不会产生不可控连锁。
- 临时食物 5 秒后消失。
- 超级药水生成的临时食物不会是 `superPotion`。

### 效果

- 加速持续 10 秒。
- 减速持续 10 秒。
- 超级药水持续 10 秒。
- 超级药水只降低非 completed 面护栏。
- 效果结束后恢复正常速度和护栏状态。
- 多个效果同时存在时有可预测的优先级。

### 关卡

- 进入新面会完成上一面。
- completed 面保持关闭。
- 新面初始长度等于上一面初始长度 +1。
- 死胡同状态能被识别但不立即判负。
- 完成立方体 6 个面后胜利。

## 8. 集成测试计划

- 设置页能渲染 MVP 控件。
- 开始游戏会创建新游戏状态。
- 游戏页渲染画布和 HUD。
- 暂停会停止游戏更新。
- 重新开始会重置游戏状态。
- 吃到食物后 HUD 长度更新。
- Active effects 会显示在 HUD。
- 死亡后显示失败状态。
- 完成立方体 6 个面后显示胜利状态。

## 9. 浏览器验收计划

使用 in-app Browser 在重要前端里程碑后检查：

- Vite 本地页面可打开。
- Canvas 非空白。
- 初始加载无控制台 error。
- `A`、`D`、`W` 操作响应正常。
- 蛇移动是连续的，不是格子跳动。
- 摄像机是俯视跟随。
- 食物可见且可吃。
- 跨面动画可读。
- 已完成面护栏保持升起。
- 超级药水会生成临时食物并正确清理。

## 10. 性能验收

目标：

- 桌面浏览器接近 60 FPS。
- 超级药水期间不明显卡顿。
- 临时食物能及时清理，避免对象堆积。
- 多次重新开始后不残留旧渲染对象或事件监听器。

## 11. MVP 完成标准

MVP 完成时应满足：

- 立方体模式可以从开始玩到胜利或死亡。
- 游戏使用连续车辆式移动。
- 默认自动前进，`A/D` 转向，`W` 加速。
- 俯视跟随摄像机可用。
- 7 种食物全部实现。
- 已完成面永久关闭。
- 路线选择可能形成死胡同。
- 设置页可以配置主要玩法参数。
- `npm run build` 通过。
- 核心逻辑单元测试通过。
