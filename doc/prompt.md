# 分 Session 开发 Prompt

本文件用于后续分阶段开发。每个 session 开始前，先复制对应 prompt 给开发 agent，或由 Codex 直接读取本文件执行。

所有 session 通用要求：

- 使用中文沟通。
- 开发前读取 `doc/requirements.md` 和 `doc/development-and-test-plan.md`。
- 开发前检查 `git status --short`。
- 不要覆盖用户已有改动。
- 核心游戏规则必须写在纯 TypeScript 模块中，不要塞进 Vue 组件或 Three.js 渲染类。
- Three.js 渲染层只负责表现、动画和输入输出连接。
- 每个阶段结束时运行可行的测试和 `npm run build`。
- 有前端变化时启动本地 dev server，并用 in-app Browser 验收 canvas、控制台和关键交互。
- 如果发现需求需要调整，先更新 `doc/requirements.md` 或 `doc/development-and-test-plan.md`。

## Session 1：Phase 1 单面可玩原型

```text
请按照 `doc/requirements.md` 和 `doc/development-and-test-plan.md` 开始 Phase 1：单面可玩原型开发。

目标：
在当前 Vue 3 + Vite + TypeScript 项目中，安装必要依赖并实现一个可玩的单面 3D 贪吃蛇原型。蛇在一个平面上连续自由移动，默认自动前进，A/D 转向，W 临时加速。玩家可以吃普通食物增长长度，撞到自身后失败。

必须完成：
1. 安装 `three` 和 `vue-router@4`。如要写测试，安装 `vitest jsdom @vue/test-utils`。
2. 建立基础目录结构：`src/game/engine`、`src/game/render`、`src/types`、`src/views`。
3. 实现纯逻辑模块：
   - 游戏状态初始化。
   - 自动前进移动。
   - A/D 转向，W 加速。
   - 基于 delta time 的移动更新。
   - 蛇身体历史轨迹跟随。
   - 普通食物生成和距离碰撞。
   - 蛇长度增长。
   - 自身碰撞，忽略头部后方前 20% 身体长度。
4. 实现 Three.js 单面场景：
   - 非空白 canvas。
   - 活动面。
   - 蛇头和身体。
   - 普通食物。
   - 基础俯视摄像机。
5. 实现基础游戏页：
   - HUD 显示长度、速度、状态。
   - 暂停、重新开始、失败状态。
6. 添加必要的单元测试，优先测试移动、食物、长度和自身碰撞。
7. 运行 `npm run build`。
8. 启动 dev server，用 in-app Browser 验收：页面可打开、canvas 非空、控制台无 error、A/D/W 可操作。

不要做：
- 不要实现跨面。
- 不要实现全部 7 种食物。
- 不要做复杂美术。
- 不要泛化正多面体。
```

## Session 2：Phase 2 立方体拓扑和跨面

```text
请按照 `doc/requirements.md` 和 `doc/development-and-test-plan.md` 开始 Phase 2：立方体拓扑和跨面开发。

目标：
在已有单面原型基础上，实现立方体 6 个面的关卡流程。蛇达到目标长度后可通过开放边进入相邻面；上一面永久 completed 并关闭；路线选择可能形成死胡同，但不立即判负。

必须完成：
1. 定义立方体拓扑：
   - 6 个面。
   - 每个面的法线。
   - 每条边的相邻面。
   - 跨边后的局部坐标映射。
   - 跨边后的朝向映射。
2. 实现面状态：
   - available。
   - active。
   - completed。
3. 实现护栏系统：
   - 未开放边护栏升起。
   - 达到本关目标长度后，可用出口护栏降下。
   - completed 面所有护栏永久升起。
4. 实现跨面：
   - 蛇穿过开放边后进入相邻面。
   - 上一面变 completed。
   - 新面蛇长度重置。
   - 新面初始长度 = 上一面初始长度 +1。
5. 实现死胡同识别：
   - 可以在 HUD 或 debug 中显示，但不立即判负。
6. 实现胜利条件：
   - 立方体 6 个面全部 completed 后胜利。
7. 更新 Three.js 渲染：
   - 显示立方体。
   - 当前面高亮。
   - completed 面变暗或锁定。
   - 护栏状态可见。
8. 更新摄像机：
   - 跨面后俯视跟随新面。
9. 添加或更新单元测试：
   - 面状态。
   - 跨面映射。
   - completed 面不可返回。
   - 长度重置和 +1。
   - 胜利条件。
10. 运行测试和 `npm run build`，并用浏览器验收跨面流程。

不要做：
- 不要扩展其他正多面体。
- 不要实现超级药水。
- 不要把跨面规则写死在 Vue 组件中。
```

## Session 3：Phase 3 食物和效果系统

```text
请按照 `doc/requirements.md` 和 `doc/development-and-test-plan.md` 开始 Phase 3：完整食物和效果系统开发。

目标：
实现 7 种食物和全部效果：普通、大餐、加速、减速、集束、彩虹糖、超级药水。

必须完成：
1. 实现食物类型和权重：
   - 普通 70%。
   - 大餐 10%。
   - 加速 5%。
   - 减速 5%。
   - 集束 5%。
   - 彩虹糖 3%。
   - 超级药水 2%。
2. 实现普通和大餐长度增长。
3. 实现加速：
   - 速度 1.5 倍。
   - 持续 10 秒。
4. 实现减速：
   - 速度 0.5 倍。
   - 持续 10 秒。
5. 实现集束：
   - 爆开生成 3 到 5 个随机食物。
   - 避免不可控连锁。
6. 实现彩虹糖：
   - 获得一次 rainbowShield。
   - 持续 8 秒或抵消一次死亡。
   - 不能进入 completed 面。
7. 实现超级药水：
   - 持续 10 秒。
   - 除 completed 面外，所有护栏降下。
   - 速度 1.5 倍。
   - 所有非 completed 面持续生成随机临时食物。
   - 临时食物 5 秒后消失。
   - 临时食物不能是 superPotion。
   - 效果结束后恢复正常护栏状态。
8. 更新渲染：
   - 不同食物有不同颜色或简单造型。
   - Active effects 在 HUD 中可见。
9. 添加或更新测试：
   - 权重生成。
   - 效果持续时间。
   - 临时食物过期。
   - 超级药水不打开 completed 面。
   - 彩虹糖只抵消一次。
10. 运行测试和 `npm run build`，并用浏览器验收食物和效果。

不要做：
- 不要开始复杂美术重做。
- 不要改变已定的概率，除非先更新需求文档。
```

## Session 4：Phase 4 摄像机、UI 和设置页

```text
请按照 `doc/requirements.md` 和 `doc/development-and-test-plan.md` 开始 Phase 4：摄像机、UI 和设置页开发。

目标：
让游戏可配置、可理解，并补齐页面流程。

必须完成：
1. 添加 Vue Router。
2. 实现设置页：
   - 多面体类型，MVP 只启用立方体，其他显示 coming soon 或 disabled。
   - 初始速度。
   - 转向速度。
   - 初始蛇长度。
   - 每完成一面后的初始长度增量。
   - 食物生成频率。
   - 超级药水临时食物生成频率。
   - 显示调试信息。
   - 音效开关。
3. 设置页开始游戏后进入游戏页，并使用当前配置创建新游戏。
4. 完善 HUD：
   - 当前面。
   - 当前关卡。
   - 当前长度。
   - 目标长度。
   - 当前速度。
   - Active effects。
   - 暂停、继续、重新开始、返回设置。
5. 完善俯视跟随摄像机：
   - 活动面始终清晰。
   - 跨面转场平滑。
   - 轻微倾斜但不影响操控判断。
6. 添加集成测试或组件测试：
   - 设置页渲染。
   - 设置影响新游戏。
   - HUD 与游戏状态一致。
7. 运行测试和 `npm run build`，并用浏览器验收设置页到游戏页流程。

不要做：
- 不要启用其他正多面体。
- 不要牺牲游戏逻辑模块边界来写 UI。
```

## Session 5：Phase 5 可爱视觉和手感优化

```text
请按照 `doc/requirements.md` 和 `doc/development-and-test-plan.md` 开始 Phase 5：可爱视觉和手感优化。

目标：
在 MVP 功能稳定后，把游戏打磨成可爱、鲜艳、柔和、不刺眼的完整体验。

必须完成：
1. 统一柔和糖果色调。
2. 将蛇表现为圆润身体。
3. 让 7 种食物在颜色和轮廓上容易区分。
4. 护栏做成玩具积木或柔软栏杆风格。
5. 添加护栏升降动画。
6. 添加彩虹糖和超级药水的柔和发光或粒子效果。
7. 添加轻盈的胜利和失败反馈。
8. 优化移动手感：
   - 转向速度。
   - W 加速倍率。
   - 摄像机跟随平滑度。
9. 检查性能：
   - 超级药水期间不卡顿。
   - 临时食物清理正常。
   - 重开多局不残留旧对象或事件监听器。
10. 运行 `npm run build`，并用浏览器做视觉和交互验收。

不要做：
- 不要引入刺眼霓虹或暗黑恐怖风。
- 不要让粒子遮挡玩法判断。
- 不要为了美术效果破坏规则可读性。
```
