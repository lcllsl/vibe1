# 3D 多面体贪吃蛇需求说明书

## 1. 产品目标

制作一个基于 Vue 3 和 Three.js 的 3D 贪吃蛇游戏。游戏保留经典贪吃蛇“移动、吃食物、变长、撞到失败”的核心乐趣，但不使用传统格子移动，而是让蛇在正多面体表面进行连续自由移动。

第一版只实现立方体。其他正多面体在立方体的移动、跨面、护栏和关卡系统稳定后再扩展。

## 2. 核心玩法

蛇一次只在一个多面体面上活动。每个面视为一关。

- 蛇使用连续空间移动，不使用格子坐标。
- 操作方式类似开车：自动前进、左右转向、按键加速。
- 蛇吃到食物后变长。
- 当前蛇长度达到本关目标长度后，可用出口护栏降下。
- 玩家可以通过已打开的边界进入相邻面。
- 一旦离开某个面，该面永久完成并关闭。
- 已完成面不可返回。
- 撞到已完成面的永久护栏会死亡，除非彩虹糖护盾抵消本次致命碰撞。

由于已完成面永久关闭，玩家进入新面的顺序会影响后续路线。路线规划不当会形成死胡同。

## 3. 操作方式

默认操作：

- `A`：向左转。
- `D`：向右转。
- `W`：临时加速。

MVP 采用自动前进模式：

- 蛇始终向当前朝向前进。
- `A` 和 `D` 改变蛇头朝向。
- `W` 让蛇临时加速，而不是控制是否前进。
- 蛇不能瞬间反向。
- 蛇身体沿头部历史轨迹跟随。
- 蛇长度使用连续路径长度表示，不使用格子数量。

## 4. 移动和身体规则

蛇头状态由当前面、局部坐标、朝向角度和速度组成。

身体表现：

- 身体由等距圆润段组成。
- 每个身体段从蛇头历史轨迹中采样。
- 蛇长度越长，身体段数量越多或段间距离保持一致。
- 自身碰撞检测应忽略靠近蛇头的一小段身体，建议忽略头部后方前 `20%` 身体长度，避免刚转弯时误判。

速度规则：

- 基础速度由设置页配置。
- 每进入新面，难度略微提升，可通过基础速度小幅提高实现。
- 加速、减速、超级药水等效果通过倍率影响当前速度。

## 5. 摄像机

摄像机使用俯视跟随视角。

- 摄像机跟随当前活动面。
- 摄像机沿当前面法线方向俯视。
- 可保留轻微倾斜，让玩家感受到 3D 多面体结构。
- 当前活动面必须清晰、可读、占据主要视野。
- 蛇跨面时，摄像机平滑旋转到新面的俯视方向。

## 6. 关卡和面规则

每个面是一关。

面状态：

- `available`：尚未完成，未来可能进入。
- `active`：当前正在游玩的面。
- `completed`：已经通过，永久关闭。

进入新面时：

- 上一面变为 `completed`。
- 上一面的所有护栏永久升起。
- 新面变为 `active`。
- 蛇长度重置。
- 新面的初始长度 = 上一面的初始长度 + 1 个连续长度单位。
- 难度轻微提升。

通关开门条件：

- MVP 使用“当前蛇长度达到本关目标长度”作为护栏降下条件。
- 目标长度可由初始长度、当前关卡序号和设置参数计算。

死胡同规则：

- 如果当前面剩余出口都通向已完成面，玩家会进入无解局面。
- 游戏不在发现死胡同时立即判负。
- 玩家最终撞到关闭护栏或已完成面护栏时死亡。
- 超级药水不能打开已完成面的护栏。
- 彩虹糖只能抵消一次死亡，不能允许玩家返回已完成面。

胜利条件：

- 完成立方体全部 6 个面。

失败条件：

- 蛇头撞到自身身体。
- 蛇头撞到升起的护栏。
- 蛇头尝试进入已完成面。
- 玩家进入死胡同后最终撞到不可通行边界。

## 7. 出口提示

默认开启轻量出口提示：

- 可通行出口使用柔和发光或颜色提示。
- 已完成面的永久护栏明显升起并降低饱和度。
- 不直接提示最优路线。
- 不自动警告玩家“这里会形成死胡同”，保留策略判断空间。

## 8. 食物类型

食物按权重随机生成。

| 类型 | 概率 | 效果 |
| --- | ---: | --- |
| 普通 | 70% | 增加少量长度。 |
| 大餐 | 10% | 大幅增加长度。 |
| 加速 | 5% | 速度变为 1.5 倍，持续 10 秒。 |
| 减速 | 5% | 速度变为 0.5 倍，持续 10 秒。 |
| 集束 | 5% | 爆开生成多个随机食物。 |
| 彩虹糖 | 3% | 获得一次临时防撞护盾。 |
| 超级药水 | 2% | 触发临时全地图高能效果。 |

食物碰撞使用蛇头与食物中心的距离检测。

## 9. 食物效果

### 普通

- 增加少量蛇长度。
- 视觉建议：糖豆、莓果或果冻小球。

### 大餐

- 增加更多蛇长度。
- 建议数值：普通食物的 3 倍。
- 视觉建议：纸杯蛋糕、甜甜圈或小蛋糕。

### 加速

- 速度倍率变为 `1.5`。
- 持续 10 秒。
- 视觉建议：闪电糖果。

### 减速

- 速度倍率变为 `0.5`。
- 持续 10 秒。
- 视觉建议：棉花糖或睡眠糖果。

### 集束

- 爆开生成多个随机食物。
- 建议数量：3 到 5 个。
- 生成位置必须是有效表面位置。
- 集束生成的新食物不应继续生成集束，避免连锁过强。
- 视觉建议：糖果礼花。

### 彩虹糖

- 获得 `rainbowShield` 效果。
- 持续 8 秒，或抵消一次致命碰撞后消失。
- 可抵消自身碰撞或护栏碰撞。
- 不能允许蛇进入已完成面。
- 如果抵消已完成面护栏碰撞，应把蛇弹回当前有效面区域。
- 视觉建议：彩虹棒棒糖或旋涡糖。

### 超级药水

- 出现概率：2%。
- 持续 10 秒。
- 除已完成面的永久护栏外，所有护栏全部降下。
- 蛇速度变为 `1.5` 倍。
- 全地图持续生成随机临时食物。
- “全地图”指所有非 `completed` 面。
- 临时食物存在 5 秒后消失。
- 临时食物不能是超级药水。
- 效果结束后，护栏恢复当前关卡正常状态。
- 已完成面护栏永远不受超级药水影响。
- 视觉建议：发光药水瓶和柔和粒子。

## 10. 设置页

设置页包含：

- 多面体类型。MVP 只启用立方体，其他选项显示为不可用或 `coming soon`。
- 初始速度。
- 转向速度。
- 初始蛇长度。
- 每完成一面后的初始长度增量。
- 食物生成频率。
- 超级药水临时食物生成频率。
- 是否显示调试信息。
- 是否开启音效。

未来可扩展的正多面体：

- 正四面体，4 面。
- 立方体，6 面。
- 正八面体，8 面。
- 正十二面体，12 面。
- 正二十面体，20 面。

## 11. 美术风格

整体风格：可爱、鲜艳、柔和、不刺眼。

设计方向：

- 柔和糖果色。
- 明亮但不过度饱和。
- 圆润蛇身。
- 低多边形玩具感多面体表面。
- 护栏像玩具积木或柔软栏杆。
- 食物通过颜色和轮廓清晰区分。
- 死亡反馈轻盈、非暴力，例如星星、弹开、柔和爆散。

避免：

- 刺眼霓虹色。
- 暗黑恐怖风。
- 尖锐金属刺。
- 过度遮挡玩法的粒子效果。

## 12. 技术数据模型草案

```ts
type PolyhedronType =
  | 'cube'
  | 'tetrahedron'
  | 'octahedron'
  | 'dodecahedron'
  | 'icosahedron'

type FaceStatus = 'available' | 'active' | 'completed'

type FoodType =
  | 'normal'
  | 'feast'
  | 'speedUp'
  | 'slowDown'
  | 'cluster'
  | 'rainbowCandy'
  | 'superPotion'

interface GameSettings {
  polyhedronType: PolyhedronType
  initialSpeed: number
  turnSpeed: number
  initialSnakeLength: number
  lengthBonusPerFace: number
  foodSpawnIntervalMs: number
  superPotionSpawnIntervalMs: number
  showDebug: boolean
  soundEnabled: boolean
}

interface SnakeHead {
  faceId: number
  position: { x: number; y: number }
  headingAngle: number
  speed: number
}

interface SnakeSegment {
  faceId: number
  position: { x: number; y: number }
}

interface FoodItem {
  id: string
  faceId: number
  position: { x: number; y: number }
  type: FoodType
  temporary: boolean
  expiresAt?: number
}

interface FaceState {
  id: number
  status: FaceStatus
  loweredEdges: string[]
}

interface ActiveEffect {
  type: 'speedUp' | 'slowDown' | 'rainbowShield' | 'superPotion'
  expiresAt: number
}
```
